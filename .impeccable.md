# Impeccable Design Context — Modern Operating Systems

This file is the source of truth for design decisions on the *Modern
Operating Systems* textbook web edition. Any impeccable skill (or any
agent doing visual work) should read this before touching the theme.

## Design Context

### Users

Senior undergraduate and graduate CS students, plus practicing engineers
who want to understand *why* their containers get throttled or their p99
spikes. They read this on a laptop or external monitor for **sustained
sessions** — chapter-length passes, often interleaved with running labs
in a terminal next to the browser. They return to specific passages
repeatedly while working, so layout stability and findability matter as
much as first-read pleasure.

The job-to-be-done is *learning a hard subject from a single trusted
source* — not skimming, not scanning, not searching for a snippet. Every
design decision should serve sustained reading first.

### Brand Personality

Three words: **curious, exploratory, playful** — held inside a
**classical scholarly** container.

That tension is the design. The form is restrained, careful, almost
quiet — Bringhurst / MIT Press / Knuth TAOCP lineage. The voice
*inside* the form is the opposite: it asks questions, follows hunches,
invites the reader into the investigation. The restraint of the
container is what makes the voice land. A scrappy container would make
the voice feel chaotic; a serious voice in a serious container is dull
academic prose.

Emotional goal: the reader should feel **trusted and welcomed** — like
a colleague handed them a beautifully-set notebook full of real
investigations, not a textbook lecturing at them.

### Aesthetic Direction

**Lineage**: classical scholarly press. Source Serif 4 body on a paper-
tone (#fbfaf6) background. Navy (#1f3a66) for primary accents, muted
oxblood (#6b3f1d) for secondary highlights. Booktabs-style tables
(top + bottom rule, header rule, no zebra). Fleuron section breaks.
Drop cap on the chapter opening paragraph. Footnote-style marginalia
where the layout permits.

**Voice cues** (the playful/curious side):
- Callouts that ask questions, not just announce facts.
- Asides that show the author's reasoning ("the surprising part is…").
- Diagrams that occasionally break the grid for emphasis.
- Section breaks (fleurons) that signal "pause, breathe, the next part
  shifts gears."

**Default theme**: **light, paper tone**. Dark theme exists but is a
secondary mode, not the default. Specifically *not* the AI-era
dark-with-cyan-glow look.

**Anti-references** — this must explicitly NOT look like:
1. **Generic dev-docs (Docusaurus, GitBook, Mintlify, ReadtheDocs)**.
   Sans-serif body, dense left sidebar, blue links, emoji-prefixed
   callouts. The fingerprint of "we configured a documentation tool."
2. **AI-era dark-with-glow**. Cyan-on-near-black, purple-to-blue
   gradients, neon accents. Looks "cool" without earning it.
3. **Old-school academic drab**. Times New Roman, single dense column,
   no imagination. Worthy but boring.
4. **Maximalist startup landing**. Hero gradients, animated counters,
   card grids, icons-above-every-heading.

**Anchor references** (positive):
- Edward Tufte — *Visual Display of Quantitative Information* — for
  marginalia, figure captions, and the discipline of "ink that earns
  its place."
- Robert Bringhurst — *The Elements of Typographic Style* — for the
  measure, leading, and respect for reading rhythm.
- MIT Press / Stripe Press print editions — for restrained navy/oxblood
  palettes and confident chapter openings.
- Donald Knuth — *The Art of Computer Programming* — for the "scholarly
  but playful asides" voice in the body text.

### Design Principles

1. **The reading column is sacred.** Sustained reading is the primary
   job. Every decision is judged first by whether it helps a reader
   stay in the prose for 30+ minutes. Layout, line length, leading,
   contrast, and font size all serve this. Decoration that breaks the
   reading rhythm is removed even if it looks good.

2. **One left edge.** Prose, tables, code, figures, captions, and
   callouts all share the same left edge of the reading column. No
   per-element auto-centering, no stair-stepped indents. The eye should
   return to the same column on every line.

3. **Restrained container, expressive voice.** The CSS is quiet so the
   prose can be loud. Bold visual moves — color shifts, pull quotes,
   marginalia, illustrations — are reserved for moments that earn them.
   The default state of any element is "calm".

4. **Print-quality typography on screen.** Real serif body (Source
   Serif 4), proper old-style numerals, hyphenation, balanced widows
   and orphans, fluid clamp-based scale that tracks viewport width.
   Headings and tables follow a single consistent type scale; nothing
   is sized by feel.

5. **Honest evidence over visual flourish.** This is a measurement-
   driven book. Tables, plots, and code are first-class content, not
   decoration to break up text. Charts show data; they are not
   sparklines used as ornament. Code blocks are typeset with the same
   care as prose.

6. **No defaults.** Every visible element has been chosen. mdBook's
   defaults, generic doc-site conventions, AI-era visual tics, and
   academic-template drabness are all explicitly rejected. If something
   in the rendered output looks like it could have come from any other
   site, treat it as a bug.
