# Lab 2 (Route B): PagerUSE — Oncall debugging with a simulated terminal

Route B is the **default** Lab 2 for Week2B.

Students see:
- a web UI that mimics a pager/oncall incident dashboard (active alerts + context)
- a simulated terminal where they can run a **restricted** set of Linux observation commands

Behind the scenes, the terminal output is generated by:
- `LLM_MODE=mock`: deterministic, scenario-provided outputs (recommended for grading/reproducibility)
- or `LLM_MODE=openai` (default): an OpenAI-compatible Chat Completions backend (optional)

The goal is to apply the **USE method** (Utilization / Saturation / Errors) and build an evidence chain:

> symptom → OS resource hypothesis → commands → observations → mechanism → mitigation + verification

A small local web app:

- `server.js`: Node.js server (Express)
- `static/`: single-page UI
- `scenarios/`: scenario definitions (JSON)

## Quick start

```bash
cd week2B/lab2_pageruse
npm install
LLM_MODE=mock npm start
```

Open http://localhost:3000

## Safety & scope

- The terminal **does not execute** commands on the host machine.
- Input is validated by a strict allowlist + regex.
- For LLM mode, the model receives only: scenario JSON + the validated command + a short instruction prompt.

## Scenario difficulty

- Easy: one dominant bottleneck, clean signals
- Medium: multiple contributing factors; students must disambiguate
- Hard: misleading symptoms, confounders, and partial observability

See `scenarios/`.
